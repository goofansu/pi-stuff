# GitHub

A reference map for interacting with GitHub resources locally and via CLI.

## Reference Map

| Resource | Sub-document | When to load |
|----------|-------------|--------------|
| Issues   | [issue.md](./references/issue.md) | Viewing, searching, or commenting on issues |

## Quick Rules

- **Always pull before reading** — fetch the resource locally first, then read from disk.
- **Read incrementally** — use small chunks; stop as soon as you have the information you need.
- **Use `gh` for writes** — creating, editing, closing resources always uses the `gh` CLI directly.
